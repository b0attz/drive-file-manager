# Product

## Register

product

## Users

Public users who need to manage files on Google Drive through a lightweight, self-hosted web interface. They want fast, direct access to their files without the weight of Google Drive's native UI or heavier third-party tools.

## Product Purpose

A single-page web app that connects to Google Drive via OAuth 2.0 and lets users upload, search, preview, share, organize, and delete files. The value is speed and simplicity: a focused tool that does file management well without unnecessary features or interface bloat.

## Brand Personality

Clean, efficient, minimal.

## Anti-references

- Not bloated: avoids the layering and settings sprawl of tools like Notion's admin UI.
- Not cold: avoids the sterile, developer-first aesthetic of GitHub's interface. Warmth through subtle tone, not decoration.

## Design Principles

1. **Do one thing well.** Every element serves file management. If it doesn't help find, move, or act on a file, it doesn't belong.
2. **Fast by default.** Interface responses feel instant. Loading states exist but rarely appear.
3. **Respect the user's files.** File names, sizes, and dates are the primary information hierarchy. Icons support but don't replace text.
4. **Accessible without compromise.** WCAG 2.1 AA compliance is a baseline, not a stretch goal. Keyboard navigation, screen reader support, and sufficient contrast are built in.

## Accessibility & Inclusion

WCAG 2.1 Level AA. Keyboard-navigable interface with visible focus states. Screen reader friendly with proper ARIA labels. Sufficient color contrast ratios. Reduced motion support where applicable.
